# 要件定義

本書は、LocalGitLabGateway の初期要件を定義する仕様書である。

## 1. 目的

<!-- sec_id: sec_lgg_req_001 -->

LocalGitLabGateway は、ChatGPT が Public Gateway Host を経由して、ファイアウォール内部に配置された Private GitLab Host 上の GitLab を安全に操作できる環境を提供することを目的とする。

## 2. 用語定義

<!-- sec_id: sec_lgg_req_002 -->

| 用語 | 定義 |
| --- | --- |
| Public Gateway Host | インターネットからアクセス可能な Linux サーバー。 |
| Public Gateway Service | Public Gateway Host 上で動作する API サービス。 |
| Private GitLab Host | ファイアウォール内部に配置された Linux サーバー。 |
| Private Bridge Agent | Private GitLab Host 上で動作するエージェント。 |
| Managed GitLab | Private GitLab Host 上で動作する GitLab。 |
| ChatGPT Operation Profile | ChatGPT に与える接続・操作情報セット。 |

## 3. システム構成

<!-- sec_id: sec_lgg_req_003 -->

```text
ChatGPT
  ↓ HTTPS
Public Gateway Service
  ↓ WireGuard VPN
Private Bridge Agent
  ↓ GitLab REST API v4
Managed GitLab
```

## 4. Public Gateway Service 要件

<!-- sec_id: sec_lgg_req_004 -->

### 4.1 基本要件

- HTTPS 通信を提供すること
- ChatGPT からの要求を受け付けること
- 要求内容を検証すること
- Private Bridge Agent へ転送すること
- 操作監査ログを記録すること

### 4.2 セキュリティ要件

- GitLab Personal Access Token を保持しないこと
- API Key による認証を提供すること
- request_id を全要求へ付与すること
- 許可された操作のみ実行可能とすること

## 5. Private Bridge Agent 要件

<!-- sec_id: sec_lgg_req_005 -->

### 5.1 基本要件

- Public Gateway Service と安全に通信できること
- Managed GitLab の REST API を代理実行できること
- GitLab 認証情報を内部保持できること
- 実行結果を Public Gateway Service へ返却できること

### 5.2 セキュリティ要件

- 外部から直接アクセス不可であること
- 許可済み Public Gateway Service からのみ通信を受け付けること
- GitLab 認証情報を外部送信しないこと

## 6. ChatGPT Operation Profile 要件

<!-- sec_id: sec_lgg_req_006 -->

### 6.1 含める情報

- Public Gateway Service URL
- API Key
- 利用可能 API 一覧
- 許可操作一覧
- 禁止操作一覧
- 操作例
- エラー時ルール

### 6.2 初期対応 API

- Project List
- Branch List
- File Get
- File Create
- File Update
- Commit Create
- Merge Request Create
- Issue Create

## 7. 初期除外範囲

<!-- sec_id: sec_lgg_req_007 -->

- GitLab 管理者 API
- User 管理
- Runner 管理
- Secret 一覧取得
- CI/CD Variable 管理
- GitLab DB 直接操作

<!-- sec_id: sec_lgg_req_008 -->
