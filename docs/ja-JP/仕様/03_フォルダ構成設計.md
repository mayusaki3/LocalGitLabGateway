# フォルダ構成設計

本書は、LocalGitLabGateway のリポジトリ内フォルダ構成を定義する仕様書である。

## 1. 目的

<!-- sec_id: sec_lgg_folder_001 -->

本書は、LocalGitLabGateway のリポジトリ内フォルダ構成を定義する。

LocalGitLabGateway は、Public Gateway Service、Private Bridge Agent、ChatGPT Operation Profile、HLDocS 準拠ドキュメント、テストを同一リポジトリで管理する。

## 2. 基本方針

<!-- sec_id: sec_lgg_folder_002 -->

- Public Gateway Service と Private Bridge Agent は責務を分離する
- 共通処理は `common` に集約する
- ChatGPT に渡す情報セットは `profiles` に配置する
- ドキュメントは `docs/ja-JP` に配置する
- テストは `tests` に配置する
- 実行環境別の設定例は `config` に配置する
- systemd、nginx、WireGuard などの配置例は `deploy` に配置する
- 機密情報はリポジトリに含めない

## 4. `src/local_gitlab_gateway` 構成

<!-- sec_id: sec_lgg_folder_004 -->

### 4.1 `public_gateway`

`public_gateway` は Public Gateway Service の実装を配置する。

```text
src/local_gitlab_gateway/public_gateway/
├── __init__.py
├── main.py
├── api/
├── auth/
├── proxy/
├── audit/
└── settings.py
```

| パス | 役割 |
| --- | --- |
| `main.py` | FastAPI アプリケーション起動点 |
| `api/` | ChatGPT 向け公開 API ルーティング |
| `auth/` | API Key 認証、認可処理 |
| `proxy/` | Private Bridge Agent への転送処理 |
| `audit/` | 監査ログ出力処理 |
| `settings.py` | Public Gateway Service 設定読み込み |

### 4.2 `private_bridge`

`private_bridge` は Private Bridge Agent の実装を配置する。

```text
src/local_gitlab_gateway/private_bridge/
├── __init__.py
├── main.py
├── api/
├── auth/
├── gitlab/
└── settings.py
```

| パス | 役割 |
| --- | --- |
| `main.py` | FastAPI アプリケーション起動点 |
| `api/` | Public Gateway Service から呼び出される内部 API |
| `auth/` | 内部 API 認証処理 |
| `gitlab/` | Managed GitLab REST API v4 呼び出し処理 |
| `settings.py` | Private Bridge Agent 設定読み込み |

### 4.3 `common`

`common` は両サービスで共有する処理を配置する。

```text
src/local_gitlab_gateway/common/
├── __init__.py
├── models/
├── errors/
├── logging/
├── middleware/
└── security/
```

| パス | 役割 |
| --- | --- |
| `models/` | 共通 Pydantic モデル |
| `errors/` | 共通例外、エラー応答定義 |
| `logging/` | 構造化ログ共通処理 |
| `middleware/` | 共通 FastAPI middleware |
| `security/` | トークン検証、ハッシュ処理など |

## 5. `profiles` 構成

<!-- sec_id: sec_lgg_folder_005 -->

`profiles` は ChatGPT Operation Profile を配置する。

```text
profiles/chatgpt/
├── operation_profile.template.md
└── operation_profile.example.md
```

| パス | 役割 |
| --- | --- |
| `operation_profile.template.md` | ChatGPT に渡す情報セットのテンプレート |
| `operation_profile.example.md` | ダミー値を使用した記入例 |

## 6. `config` 構成

<!-- sec_id: sec_lgg_folder_006 -->

`config` は設定ファイル例を配置する。

```text
config/
├── public_gateway/
│   └── config.example.yaml
└── private_bridge/
    └── config.example.yaml
```

設定ファイル例には機密値を含めない。

実運用の設定ファイルは、サーバー上にのみ配置し、Git 管理しない。

## 7. `deploy` 構成

<!-- sec_id: sec_lgg_folder_007 -->

`deploy` はデプロイ補助ファイルの例を配置する。

```text
deploy/
├── systemd/
│   ├── local-gitlab-gateway-public.service.example
│   └── local-gitlab-gateway-private.service.example
├── nginx/
│   └── local-gitlab-gateway.conf.example
└── wireguard/
    ├── public-gateway-wg0.conf.example
    └── private-gitlab-wg0.conf.example
```

## 10. 機密情報管理

<!-- sec_id: sec_lgg_folder_010 -->

以下はリポジトリに含めてはならない。

- GitLab Personal Access Token
- ChatGPT から Public Gateway Service へ接続する実 API Key
- Private Bridge Agent 内部 API Key
- WireGuard 秘密鍵
- TLS 秘密鍵
- 実運用サーバーの詳細な内部 IP 一覧

## 11. 初期作成対象ファイル

<!-- sec_id: sec_lgg_folder_011 -->

初期実装前に以下を作成する。

- `pyproject.toml`
- `.gitignore`
- `src/local_gitlab_gateway/public_gateway/main.py`
- `src/local_gitlab_gateway/private_bridge/main.py`
- `src/local_gitlab_gateway/common/__init__.py`
- `config/public_gateway/config.example.yaml`
- `config/private_bridge/config.example.yaml`
- `profiles/chatgpt/operation_profile.template.md`

## 12. 決定事項

<!-- sec_id: sec_lgg_folder_012 -->

- Public Gateway Service と Private Bridge Agent は同一 Python パッケージ内で分離する
- 共通モデル、共通エラー、共通ログ処理は `common` に集約する
- ChatGPT Operation Profile は `profiles/chatgpt` で管理する
- 実運用の機密設定ファイルはリポジトリに含めない
